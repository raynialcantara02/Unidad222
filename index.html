1. Qué son los Modelos de estimación?

Los Modelos de estimación son métodos y técnicas utilizados en la gestión de proyectos de software para prever la cantidad de recursos necesarios, el tiempo de desarrollo y otros parámetros relevantes antes de iniciar el proyecto.

2. Investiga sobre diferentes modelos de estimación utilizados en la industria del desarrollo de software, como el modelo COCOMO o el método de puntos de función. Compara y contrasta dos modelos de estimación y explica en qué situaciones serían más adecuados.

Los modelos de estimación como COCOMO y el método de puntos de función son utilizados para prever el esfuerzo y los recursos necesarios en proyectos de software. COCOMO es útil para proyectos grandes donde se dispone de datos históricos detallados, mientras que el método de puntos de función es más adecuado para proyectos donde se requiere una estimación temprana y precisa basada en las funciones del software.

3. A qué llamamos la estimación de un proyecto de software?

La estimación de un proyecto de software es el proceso de prever cuánto tiempo llevará desarrollar el software, cuántos recursos se necesitarán y cuál será el costo asociado. Es fundamental para la planificación y gestión efectiva del proyecto.

4. Enumera y explica brevemente los principios esenciales para el desarrollo de sistemas, como la modularidad, la cohesión, el acoplamiento y la abstracción. Elige uno de los principios y da un ejemplo de cómo se podría aplicar en el desarrollo de un sistema.

Uno de los principios esenciales es la modularidad, que consiste en dividir el sistema en módulos independientes y bien definidos. Por ejemplo, en el desarrollo de un sistema de gestión de inventario, se podría aplicar la modularidad dividiendo el sistema en módulos como gestión de productos, gestión de inventarios y gestión de pedidos, cada uno responsable de funciones específicas sin depender excesivamente de los demás.

5. Crea una lista de las partes interesadas (stakeholders) que podrían estar involucradas en un proyecto de desarrollo de software para una aplicación móvil. Para cada parte interesada, identifica sus roles, responsabilidades y posibles expectativas con respecto al proyecto.

Las partes interesadas pueden incluir: clientes (para definir requisitos y aprobar entregas), desarrolladores (para implementar y probar el software), diseñadores (para crear la interfaz de usuario), y gerentes de proyecto (para coordinar y gestionar el desarrollo).

6. Diseña un plan de proyecto en el que se definan las diferentes fases y actividades necesarias para desarrollar un sistema de gestión de inventario. Asocia cada actividad con su respectiva fase y establece las dependencias lógicas entre ellas.

El plan de proyecto incluirá fases como análisis de requisitos, diseño del sistema, desarrollo, pruebas y despliegue. Por ejemplo, la fase de análisis de requisitos incluirá actividades como la recopilación de requisitos funcionales y no funcionales, mientras que la fase de desarrollo dependerá de la finalización exitosa del diseño del sistema.

7. Elabora un conjunto de normas y estándares que deben seguirse en el desarrollo de software para garantizar la consistencia en la codificación y la documentación. Proporciona ejemplos concretos de cómo se aplicarían estas normas en la práctica.

Normas como el uso de convenciones de nomenclatura, la documentación detallada de código y la revisión por pares aseguran la consistencia. Por ejemplo, establecer una convención de nombres para variables y funciones (camelCase) y requerir que cada función tenga comentarios explicativos mejoraría la legibilidad y mantenibilidad del código.
